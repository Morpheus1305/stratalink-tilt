# Build PoLi/PoMI Market Integrity Page

## Context

This page brings together the two core verification layers of the Liquidity Truth Stack: PoLi (Proof of Liquidity, L4) and PoMI (Proof of Market Integrity, L3). PoLi verifies whether liquidity is real. PoMI is the stability protocol that would coordinate a market response when conditions deteriorate.

PoLi is operational with live data. PoMI is patent-filed and in active development, with its three-pillar framework validated against the October 2025 event data. The page must reflect both states accurately: PoLi data is live, PoMI data is derived from the same live inputs but framed as the protocol framework rather than an active intervention system.

The visual design matches the storyboard aesthetic exactly. This page is what the storyboard would look like if it were running against live markets instead of reconstructing a historical event.

| Indicator | Source | Threshold Logic |

| FUND RATE | Funding rate from perpetual venues (if available in TSLE buffer) | Green < 0.05%, Amber < 0.20%, Red >= 0.20% |

If a specific metric is not yet available from the data pipeline, show the value as `--` with a dim styling rather than inventing a number. Add a subtle tooltip or label indicating "Awaiting feed" for unavailable metrics.

### Row 2: PoLi and PoMI Cards (Side by Side)

Two cards in a horizontal 1px-gap grid, each taking 50% width.

#### Left Card: PoLi Market Integrity

```
[Score Ring 86]  PoLi Market Integrity                    PoLi Score    RATING    MARKET STATUS
                 Cross-venue liquidity integrity monitor   86           AA        EARLY STRESS

                 Description text...                                             STRATALINK
                                                                                 PoLi v1.0
```

- Score ring: 56px, stroke coloured by RAG, score number centred
- Title: "PoLi Market Integrity" in 13px sans-serif, `#D8DEE8`
- Subtitle: "Cross-venue liquidity integrity monitor" in 11px, `#7B8EA3`
- PoLi Score: Large value (22px monospace, bold, coloured by RAG) with 10px label above
- Rating: Large value (22px monospace, bold, `#D8DEE8`) with 10px label above
- Market Status: 14px monospace, bold, coloured by RAG. Derived from PoLi score and regime:
  - Score >= 85: "STABLE" (green)
  - Score >= 70: "MONITORING" (green)
  - Score >= 50: "EARLY STRESS" (amber)
  - Score >= 30: "FRAGMENTING" (amber)
  - Score < 30: "CRITICAL" (red)
- Description: 11px, `#7B8EA3`. One or two lines describing the current state based on the score and detected conditions. Generate dynamically:
  - High score: "Cross-venue liquidity conditions within normal parameters. No structural deterioration detected."
  - Medium: "Minor structural deterioration detected across tier-2 venues. Monitoring for escalation."
  - Low: "Investment-grade integrity conditions deteriorating rapidly. Cross-venue fragmentation accelerating."
- Version label: "STRATALINK PoLi v1.0" in 9px monospace, `#4A5B6E`

**Data source:** Live PoLi score from the PoLi engine (same as TILT). Rating from the PoLi rating bands.

#### Right Card: PoMI Market Integrity

```
[Score Ring 92]  PoMI Market Integrity                    PoMI Score    RATING    STATUS
                 Coordination integrity monitor            92           AAA       STABLE

                 ● Threshold [status]                     Description text...    STRATALINK
                 ● Throttle  [status]                                            PoMi v1.0
                 ● Venue Sync [status]
```

- Same layout as the PoLi card but with the three-pillar status indicators
- Three status lines showing the PoMI pillars:
  - **Threshold**: Status derived from EWDS indicator levels. All green = "Ready" (green). Any amber = "Monitoring" (amber). Any red = "Triggered" (red).
  - **Throttle**: Status derived from regime classification. NORMAL = "Standby" (green). ELEVATED = "Partial" (amber). STRESSED = "Active" (red).
  - **Venue Sync**: Status derived from cross-venue divergence. Low divergence = "Coordinated" (green). Moderate = "Minor Drift" (amber). High = "Fragmenting" (red).
- PoMI score: Compute as a weighted average of the three pillar scores (each pillar scored 0-100 based on the thresholds above). This is a framework score showing what PoMI would report, not an active intervention measurement.
- Description: Dynamic based on PoMI score:
  - High: "Coordination mechanisms intact. Stability infrastructure operational."
  - Medium: "Coordination window narrowing. Partial throttle engagement advisable."
  - Low: "Cross-venue stabilisation integrity compromised. Systemic coordination capacity constrained."
- Version label: "STRATALINK PoMi v1.0" in 9px monospace, `#4A5B6E`

**Important framing note:** PoMI scores and statuses are derived from live market data using the PoMI framework logic, but PoMI is not actively intervening in markets. The three-pillar display shows what the coordination protocol's assessment would be, based on the patent-filed framework. This distinction does not need to appear on the page itself (the verbal framing during the demo handles it), but it should be understood during implementation.

### Row 3: Execution Feasibility Index

A full-width chart panel.

```
|  EXECUTION FEASIBILITY INDEX                            ● NOMINAL   0.798
```

- Section label with teal accent bar
- Right-aligned: RAG dot + status label + numeric value in 16px monospace bold
- Chart: Line chart showing the Execution Feasibility Index over time (using the TSLE buffer's time window, approximately 1 hour of data)
- X-axis: Time labels in monospace `#4A5B6E`
- Y-axis: 0.00 to 1.00 scale
- Line colour: `#00BFA5` (accent)
- Dashed reference lines at key thresholds (e.g., 0.80 for nominal, 0.50 for stressed) in `#1A2435`
- Chart background: `#0F151F`

**Data source:** Compute the Execution Feasibility Index from the L5F factors. A simple approach: normalise the L5F composite score to a 0-1 range (divide by 100). A more sophisticated approach: weight Depth Quality and Execution Integrity more heavily since they directly affect execution feasibility.

**Status labels:**
- >= 0.80: "NOMINAL" (green)
- >= 0.50: "DEGRADED" (amber)
- >= 0.20: "NON-EXECUTABLE" (amber/red)
- < 0.20: "MARKET FAILURE" (red)

For the chart, plot the historical values from the TSLE buffer's ring buffer. Each data point in the buffer represents a 10-second snapshot over the last hour. If the buffer does not store historical composite scores, compute them from the stored L5F factor values.

If historical data is not available, show just the current value as a single point on the chart with a label, and note "Building history..." in `#4A5B6E` until sufficient data points have accumulated.

### Row 4: Cross-Venue Visualisations (Two-Column)

Left column (approximately 50%): Cross-Venue Liquidity Stability heatmap.
Right column (approximately 50%): Venue Depth and Spread Monitor table plus Liquidation Pressure Index.

#### Left: Cross-Venue Liquidity Stability Heatmap

```
|  CROSS-VENUE LIQUIDITY STABILITY
```

A grid visualisation showing venue health over time. Each column represents a venue, each row represents a time interval (e.g., last 6 snapshots). Cells are coloured:
- Green (`#00E676` at 40% opacity): Venue healthy, depth and spread within normal parameters
- Amber (`#FFB300` at 40% opacity): Venue stressed, depth declining or spread widening
- Red (`#FF5252` at 40% opacity): Venue critical, depth collapsed or spread extreme

**Data source:** Per-venue depth and spread data from the TSLE buffer, evaluated against thresholds. Use the same venue data that feeds the TILT Cross-Venue Depth Map.

```
■ STABLE (count)    ■ STRESSED (count)    ■ CRITICAL (count)
```

If historical per-venue data is not available in the TSLE buffer, show a single row (current state only) rather than a time-series grid.

- Accent: `#00BFA5`
- RAG: `#00E676` (green), `#FFB300` (amber), `#FF5252` (red)
- Typography: JetBrains Mono for all data, system sans-serif for descriptions only
- No shadows, no gradients, no border-radius > 2px
