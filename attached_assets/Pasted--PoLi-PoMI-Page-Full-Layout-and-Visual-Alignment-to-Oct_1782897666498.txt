## Section 4: Cross-Venue Liquidity Stability Heatmap (CRITICAL CHANGE)

### Current state
A flat horizontal bar-style visualisation that does not clearly map
individual venues to individual cells.

### Required state
A grid of squares — one square per venue — where the colour represents
the stability state of that specific venue in real time.

**Grid layout**: 13 venues in a 7-column x 2-row grid (or closest
reasonable fit). Each cell:
- Size: uniform squares, roughly 80x60px each
- Label: short venue ticker inside or below the square (GMX, UNI, HYP,
  KRK, OKX, DYD, OTC, COI, BIN, BGT, BYB, DER, CRV)
- Background colour based on stability state:
    GREEN (#00E676) = STABLE: venue depth score > 60 AND spread < 5bps
    AMBER (#FFB300) = STRESSED: depth 40–60 OR spread 5–15bps
    RED (#FF5252) = CRITICAL: depth < 40 OR spread > 15bps
- Border: 1px #1A2435
- Cell background when dark: #0F151F

**Legend below the grid**:
  ● STABLE (N)   ● STRESSED (N)   ● CRITICAL (N)
  where N counts the squares in each state, updating in real time.

**Critically**: Curve Finance must appear as a square in this grid. It
has been missing from the PoLi/PoMI venue list — this is the fix for
POL-06 (Replit_Fix_PoLiPoMI_Venue_Count_Mismatch.md). Ensure all 13
configured venues from shared/venue-config.ts appear as squares, not
just the 12 currently rendering.
