CONTEXT / NON-NEGOTIABLES
We are implementing Phase A of Stratalink’s Regulatory Consumption Layer (RCL): a read-only, regulator-safe UI/UX view for ADGM inside TILT.
This is NOT a trading UI. This is NOT a dashboard with knobs. This is an evidence browser / market integrity observatory.

You MUST:
- Implement the RCL API contract exactly (shape-first)
- Build one single-screen UI (“Liquidity Truth — Regulatory View”) with three panes
- Use mocked data behind the API for now (no dependency on live PoLi/DACT/LIS wiring)
- Keep everything read-only and calm; no live tickers, no charts, no order book ladders
- Include Export buttons (PDF + JSON) even if PDF is placeholder for now
- Keep code clean, minimal, and ready to swap mock providers for real services later

DO NOT:
- Add any write endpoints
- Add any features beyond what is specified
- Add trading/strategy features or “alerts optimized for response”
- Show prices, depth ladders, or anything that resembles execution

TARGET ARCHITECTURE
- Add server routes under: /api/rcl/v0.1/*
- Add UI page/route in TILT for: /regulatory/adgm (or similar)
- The UI should fetch from GET /api/rcl/v0.1/screen/adgm?instrument=BTC-USD (instrument is selectable from a dropdown)
- Use /api/rcl/v0.1/instruments to populate the dropdown
- All payloads must include: meta, access_context, header, coverage, truth, provenance, export (exact keys)

PHASE A ACCEPTANCE CRITERIA (PASS/FAIL)
PASS if:
1) GET /api/rcl/v0.1/screen/adgm returns a JSON payload matching the agreed contract keys:
   meta, access_context, header, coverage, truth, provenance, export
2) GET /api/rcl/v0.1/instruments returns list items with instrument, asset_class, status
3) UI renders a single screen with:
   - Header section: jurisdiction=ADGM, market scope, time window label, non-authoritative notice
   - Left pane (Coverage): venue_count, liquidity_types, coverage_pct, last_successful_ingest_at, coverage_flags
   - Center pane (Truth): PoLi status, evidence_level, verified_at, valid_until, integrity summary

4) No charts, no tickers, no orderbook depth, no trading language
5) Export JSON works (downloads or opens the JSON)
6) Export PDF endpoint exists and returns a placeholder PDF (simple “RCL Snapshot Placeholder” is ok)

FAIL if:
- Any data mutation endpoints exist
- UI shows prices or anything resembling a trading terminal
- Contract keys differ or fields are renamed/omitted
