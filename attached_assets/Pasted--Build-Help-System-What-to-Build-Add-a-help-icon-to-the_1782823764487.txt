# Build Help System

## What to Build

Add a help icon to the header bar in `dashboard-header.tsx`. The icon is a "?" character or a question mark circle icon from the lucide-react icon library (`HelpCircle`). Position it between the settings cog and the sign-out arrow, so the header utility icons read left to right: Bell, Reports, Settings, Help, Sign Out.

When clicked, a slide-out panel appears from the right (same pattern as the notification bell, reports panel, and settings panel) containing a self-contained support hub.

---

## Header Icon

```
[Bell 🔔]  [Reports 📄]  [Settings ⚙]  [? Help]  [→ Sign Out]
```

- Icon: `HelpCircle` from lucide-react, or a simple "?" character styled as a circle
- Size: Same as the other header icons (20px)
- Colour: `#7B8EA3` (text-secondary), shifts to `#00BFA5` (accent) on hover
- No badge count (help is not a notification system)

---

## Help Panel Structure

The panel has a left navigation list and a right content area, similar to the settings panel.

### Panel styling

- Width: 520px (slightly wider than settings to accommodate documentation content)
- Background: `#0B1019`
- Left border: 1px `#1A2435`
- Slide-in animation: 0.2s ease-out from right
- Close: X button, click outside, or Escape key
- Header: "HELP" in 12px monospace, `#00BFA5`, with close button
- Section navigation: Vertical list on the left (140px wide), 11px monospace, `#7B8EA3`, active section with `#00BFA5` text and 2px left accent bar

### Navigation sections

```
QUICK START
USER GUIDE
GLOSSARY
DATA DICTIONARY
FAQ
WHAT'S NEW
SYSTEM STATUS
CONTACT SUPPORT
```

## Contextual Help Integration

In addition to the help panel, add small "?" icons next to key metrics across all pages. These are the contextual help tooltips specified in the In-App Help Specification document.

### Implementation

- Add a reusable `HelpTooltip` component that accepts a `helpKey` string
- The component renders a small "?" icon (12px, `#4A5B6E`, hover: `#00BFA5`)
- On hover or click, a tooltip appears showing the help text for that key
- Help text is sourced from `shared/help-content.ts` using the key
- The tooltip includes a "Learn more" link that opens the Help panel to the relevant Glossary or User Guide section

### Where to place contextual help icons

- Next to the L5F Composite score label
- Next to each L5F factor name (Depth Quality, Resilience, Fragmentation, Execution Integrity, Regime Stability)
- Next to the Regime Classification label
- Next to each EWDS indicator label
- Next to each STRATA AI detection category label
- Next to the PoLi Score and PoMI Score labels
- Next to the Evidence Level label on the RCL page
- Next to the Execution Feasibility Index label
- Next to the Fragmentation Index label

Do not overdo it. Only place help icons next to terms that a new user would not immediately understand. Do not place them next to obvious labels like "Venue" or "Spread".

## Help Content Storage

```typescript
export const HELP_CONTENT = {
  quickStart: { ... },
  userGuide: {
    liquidity: {

      panel1: { title: "Liquidity Health Core", content: "..." },
      ...

    strataAi: { ... },
    poliPomi: { ... },
    rcl: { ... },
    alerts: { ... },
  },
  glossary: [
    { term: "ADL COUNT", definition: "..." },
    { term: "ALT LIQ", definition: "..." },
    ...
  ],
  dataDictionary: { ... },
  faq: { ... },
  tooltips: {
    l5f_composite: "The Liquidity 5-Factor Score...",
    depth_quality: "Measures orderbook depth...",
    ...
  },
  releaseNotes: [ ... ],
  contactSupport: { ... },

```

This file is the single source of truth for all help content. The help panel, contextual tooltips, and any future help features all consume from this file.

Populate it with the content from the attached documentation: Quick Start Guide, User Guide, Glossary, Data Dictionary, and FAQ.

## What NOT to Change

- Do not modify any existing page layouts or data logic.
- Do not modify the other header utilities (bell, reports, settings).
- Do not add a sixth tab. Help is a header icon, not a page.
- Do not duplicate content. All help text lives in `shared/help-content.ts` and is consumed from there.
