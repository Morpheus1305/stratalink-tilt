# Phase 1: Add RWA / Digital Securities Assets to TILT

## Context
This is a strategic expansion from virtual assets into tokenized
real-world assets (RWAs) and digital securities. These assets already
trade on venues Stratalink connects to - no new venue relays are needed.
The key change is adding these tokens to the monitored asset universe
so the platform can provide liquidity verification for institutional
products like BlackRock's BUIDL alongside BTC and ETH.

This directly addresses ADGM's fourth asset category (Digital Securities)
and positions Stratalink as an asset-class-agnostic liquidity
verification layer rather than a crypto-only monitoring tool.

## Strategic Significance
BlackRock's BUIDL ($2.5B tokenized Treasury fund) started trading on
Uniswap in February 2026. This means a regulated institutional product
from the world's largest asset manager is already on a DEX that
Stratalink connects to. The same PoLi scoring engine that tells you
whether BTC liquidity is real can tell you whether a tokenized Treasury
fund's on-chain liquidity is real. That is an extraordinary positioning
statement.

Add a new category in the ILU token selector alongside the existing

## Implementation

### Step 1: Extend the asset universe

Add the five new tokens to shared/ilu-universe.ts (or equivalent) under
a new "DIGITAL SECURITIES & RWA" category. The total asset count will
increase from 20 to 25.

### Step 2: Add token-to-address mappings

For each new token, add the contract addresses for every chain where
the token exists, following the same pattern used for the existing
ILU-20 tokens.

### Step 3: Extend venue relay pair mappings

For each venue relay, add pair mappings for the new tokens where they
are listed. Follow the same pattern as the ILU-20 relay extension:
- Attempt to fetch data for each new token
- Return valid data where listed
- Return null where not listed (accurate coverage)

### Step 4: Update the token selector UI

Add the new "DIGITAL SECURITIES & RWA" category header in the token
selector dropdown. The five new tokens should appear under this
category with appropriate icons/logos.

### Step 5: Verify across all tabs

For each new token, confirm:
- LIQUIDITY tab renders with available venue data
- STRATA AI detection runs against the token
- PoLi/PoMI scores and heatmap populate
- RCL shows accurate coverage (some tokens may have low venue counts -
  this is correct and honest)
- ALERTS generates depth/divergence alerts where applicable
- Footer ticker includes the new tokens

## Important Notes

### Coverage expectations
These assets will have VERY different coverage profiles from BTC/ETH:
- BUIDL: likely 1-2 venues (Uniswap only). Coverage 8-15%. This is
  accurate - BUIDL is a whitelisted institutional product, not a
  retail-accessible token on 13 exchanges.
- PAXG: likely 4-6 venues. Coverage 30-50%. Good for a commodity token.
- XAUT: likely 2-4 venues. Coverage 15-30%.
- ONDO: likely 6-8 venues. Coverage 50-60%. Well-distributed DeFi token.
- OUSG: likely 0-1 venues. Coverage 0-8%. Institutional-only product.

These coverage percentages are NOT a problem. They are honest reporting
of where these assets actually trade. The RCL page will show DEGRADED
or INSUFFICIENT for low-coverage tokens - this is correct and
demonstrates that the platform does not fabricate coverage.

### PoLi scoring adjustments
Tokenized Treasuries (BUIDL, OUSG) and gold tokens (PAXG, XAUT) behave
very differently from volatile crypto assets:
- Spreads will be much tighter (BUIDL trades at exactly $1.00)
- Depth may be lower in absolute terms but higher relative to trading
  volume
- Volatility will be near zero for pegged assets
- The L5F score should still compute correctly but the interpretation
  differs - a low Depth Quality score for BUIDL means something
  different from a low DQ score for BTC

No scoring engine changes are needed for Phase 1. The scores will
compute correctly from the raw data. Interpretation guidance can be
added to the Supervisory Analyst Field Guide later.
