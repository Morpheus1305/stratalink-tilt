# Build TILT Reporting System

## Context

TILT needs a reporting system that serves three use cases: on-demand export from the page you are on, scheduled automated reports delivered via email, and a report history for audit trail. This prompt builds all three layers.

Five report templates are attached to this prompt. They define the exact structure, sections, tables, and data fields that the generated reports must match. Use them as the definitive specification for report output.

The attached templates are:
1. **TILT_Report_Daily_Liquidity_Summary.docx** - Scheduled daily, covers all supervised tokens
2. **TILT_Report_Weekly_Integrity_Digest.docx** - Scheduled weekly, trend analysis and intelligence summary
3. **TILT_Report_Monthly_Supervisory_Overview.docx** - Scheduled monthly, executive overview
4. **TILT_Report_Token_Liquidity.docx** - On-demand, single token deep dive
5. **TILT_Report_Incident_Template.docx** - On-demand, incident documentation

---

## Architecture Overview

The reporting system has three layers:

```
Layer 1: In-page export buttons (frontend, on each data page)
Layer 2: Reports panel (slide-out panel from header, accessible on every page)
Layer 3: Backend generation service (server-side, scheduled + on-demand)
```

---

## Layer 1: In-Page Export Buttons

### What to build

Add a compact export button to the header area of each data page. The button uses a download/document icon with the text "EXPORT" in 10px monospace. Clicking it opens a small dropdown showing the available export options for that page.

### Per-page export options

**Liquidity page (`/platform/tilt`):**
- "Token Liquidity Report (PDF)" - Generates the Token Liquidity Report for the currently selected ILU-20 token using live data
- "Cross-Venue Comparison (PDF)" - Generates a venue comparison table for the current token

**STRATA AI page (`/platform/strata-ai`):**
- "Intelligence Summary (PDF)" - Generates a STRATA AI detection summary for the current token including all six detection categories, recent signals, and EWDS status

**PoLi/PoMI page (`/platform/integrity`):**
- "Verification Report (PDF)" - Generates the PoLi/PoMI assessment with venue attribution, EWDS indicators, and PoMI pillar status

**RCL page (`/regulatory/adgm`):**
- "Supervisory Snapshot (PDF)" - Generates the RCL Coverage, Truth, and Provenance data as a PDF
- "Supervisory Snapshot (JSON)" - Exports the raw RCL data payload as JSON

**Alerts page (`/platform/alerts`):**
- "Alert History Export (PDF)" - Exports recent alert records
- "Alert History Export (CSV)" - Exports alert data as CSV for analysis

### Export button styling

Follow the house style:
- Button: `bg-panel` background, 1px `border`, 2px border-radius, 10px monospace text, `text-secondary` colour
- Hover: Border shifts to `accent`
- Dropdown: `bg-panel` background, 1px `border`, subtle shadow (`0 4px 12px rgba(0,0,0,0.4)`), each option as a clickable row with hover state
- While generating: Show a brief "Generating..." state with a subtle loading indicator, then trigger the browser download

### How the export works

When the user clicks an export option:

1. The frontend calls a backend endpoint (e.g., `POST /api/reports/generate`) with the report type, token, and timestamp.
2. The backend pulls the current live data from the same sources the frontend consumes (L5F snapshot, PoLi engine, TSLE buffer, alert service, venue relay status).
3. The backend generates a PDF matching the attached template structure.
4. The PDF is returned as a download response.
5. A record of the generated report is saved to the database for the Report History.

## Layer 3: Backend Report Generation Service

### What to build

Create `server/services/reportGenerator.ts` - the service that generates PDF reports from live data.

### PDF generation approach

Use one of these approaches (check what is already available in the project dependencies):

**Option A (preferred): Use the same `docx` library (docx-js) the project already uses.** Generate a .docx file matching the template structure, then convert to PDF using a headless LibreOffice call or a docx-to-pdf library. This produces reports that match the templates exactly.

**Option B: Use `pdfkit` or `pdf-lib`.** Generate PDFs directly. This requires manually laying out each section, table, and text block, but avoids the docx-to-pdf conversion step.

**Option C: Use HTML-to-PDF.** Render the report as an HTML page using the house style, then convert to PDF using `puppeteer` or `playwright`. This leverages existing React/HTML skills and produces clean PDFs.

Choose whichever approach is fastest to implement. The critical requirement is that the output matches the section structure, table layouts, and data fields shown in the attached templates.

### Report generation functions

Create a function for each report type:

```typescript
async function generateDailyLiquiditySummary(tokens: string[], date: Date): Promise<Buffer>
async function generateWeeklyIntegrityDigest(tokens: string[], weekEnding: Date): Promise<Buffer>
async function generateMonthlySupervisoryOverview(tokens: string[], month: Date): Promise<Buffer>
async function generateTokenLiquidityReport(token: string, timestamp: Date): Promise<Buffer>
async function generateIncidentReport(incident: IncidentData): Promise<Buffer>
```

Each function:
1. Pulls live data from the TSLE buffer, L5F engine, PoLi engine, alert service, and venue relay status
2. Populates the report template structure with real values
3. Generates the PDF
4. Returns the PDF as a Buffer

### Data sources for each report

**Daily Liquidity Summary:**
- Per-token PoLi scores, ratings, depth, spread, status: from L5F snapshot endpoint for each supervised token
- EWDS indicators per token: from TSLE buffer stress metrics
- Alerts from last 24 hours: from alert service database query
- STRATA AI detection summary: from detection evaluation for each token
- Supervisory notes: auto-generated text based on which tokens are stressed or elevated

**Weekly Integrity Digest:**
- Daily PoLi scores for each day of the week: from persisted historical data (if available) or from the current snapshot with trend data
- Alert activity summary: count by severity from alert service
- STRATA AI signal counts by category: from detection signal history
- Venue performance: uptime, latency, gaps from venue relay metrics
- Key observations and recommendations: auto-generated from the data

**Monthly Supervisory Overview:**
- Month-start and month-end scores: from persisted data
- Alert statistics: from alert service
- STRATA AI monthly summary: from detection signal history
- Venue reliability: from venue relay metrics
- System performance: uptime, latency from server metrics

**Token Liquidity Report:**
- Everything from the L5F snapshot for the selected token
- PoLi score breakdown (depth, balance, spread components)
- Per-venue attribution from venue evidence service
- All six EWDS indicators with thresholds
- All six STRATA AI detection categories with status
- Data provenance from RCL data service

### Report metadata and storage

Create a database table for report records:

```sql
CREATE TABLE report_records (
  id SERIAL PRIMARY KEY,
  report_type VARCHAR(50) NOT NULL,  -- 'daily', 'weekly', 'monthly', 'token', 'incident'
  token_scope VARCHAR(255),           -- 'portfolio' or specific token symbol
  generated_at TIMESTAMP NOT NULL,
  generated_by VARCHAR(100),          -- 'scheduled' or user identifier
  delivery_status VARCHAR(20),        -- 'generated', 'delivered', 'failed'
  file_path VARCHAR(500),             -- path to stored PDF
  reference_id VARCHAR(100) NOT NULL, -- unique report reference (e.g., DLS-20260628-001)
  metadata JSONB                      -- additional data (recipients, config, etc.)
);
```

### Scheduled generation

Use `node-cron` (already in the project) to schedule report generation:

```typescript
// Daily at 07:00 UTC
cron.schedule('0 7 * * *', async () => {
  const config = await getScheduledReportConfig('daily');
  if (config.active) {
    const pdf = await generateDailyLiquiditySummary(config.tokens, new Date());
    await storeReport('daily', 'portfolio', pdf);
    await deliverReport(config.recipients, pdf, 'Daily Liquidity Summary');
  }
});

// Weekly on Fridays at 07:00 UTC
cron.schedule('0 7 * * 5', async () => { ... });

// Monthly on the 1st at 07:00 UTC
cron.schedule('0 7 1 * *', async () => { ... });
```

## API Endpoints

Create the following endpoints in a new route file `server/routes/reports.ts`:

```
POST   /api/reports/generate          - Generate an on-demand report (type, token, format)
GET    /api/reports/history            - List generated reports (paginated)
GET    /api/reports/download/:id       - Download a specific report by ID
GET    /api/reports/scheduled          - Get scheduled report configurations
PUT    /api/reports/scheduled/:type    - Update a scheduled report configuration
POST   /api/reports/incident           - Create and generate an incident report
```

All endpoints require authentication. The report generation endpoint should return the PDF as a download response for on-demand exports, and store it for scheduled reports.
