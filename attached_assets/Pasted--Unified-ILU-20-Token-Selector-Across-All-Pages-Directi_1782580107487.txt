# Unified ILU-20 Token Selector Across All Pages

## Directive

The Stratalink platform monitors the Institutional Liquidity Universe (ILU-20): 20 assets selected by systemic importance, not market capitalisation. The token selector must be unified across every page in the platform. When a user selects a token on any page, every other page updates to reflect that token. There must be only one token selection mechanism in the entire platform, and it must offer exactly the ILU-20 tokens.

Remove all existing token selectors, token dropdowns, BTC/ETH/SOL button strips, and any other token selection UI currently on any page. Replace them all with the single unified ILU-20 selector described below.

## Step 2: Create Global Token State

Create a React context that manages the selected token globally across all pages. Every page consumes this context rather than maintaining its own local token state.

Create `client/src/contexts/TokenContext.tsx`:

```typescript
import { createContext, useContext, useState, ReactNode } from "react";
import { ILU_TOKENS, type ILUToken } from "../../shared/ilu-universe";

interface TokenContextType {
  selectedToken: ILUToken;
  setSelectedToken: (token: ILUToken) => void;
  selectedSymbol: string;
  selectedPair: string;
}

const TokenContext = createContext<TokenContextType | null>(null);

export function TokenProvider({ children }: { children: ReactNode }) {
  const [selectedToken, setSelectedToken] = useState<ILUToken>(ILU_TOKENS[0]); // Default to BTC

  return (
    <TokenContext.Provider value={{
      selectedToken,
      setSelectedToken,
      selectedSymbol: selectedToken.symbol,
      selectedPair: selectedToken.pair,
    }}>
      {children}
    </TokenContext.Provider>
  );
}

export function useToken() {
  const ctx = useContext(TokenContext);
  if (!ctx) throw new Error("useToken must be used within TokenProvider");
  return ctx;
}
```

Wrap the entire application in `TokenProvider` at the top level in `App.tsx` or the root layout component, so every page has access to the selected token.

### Determining LIVE vs -- Status

Check whether the venue data pipeline is currently returning data for each token. The simplest approach: when the L5F snapshot endpoint is called for a given token's pair, does it return a non-null response with a non-zero depth? If yes, that token is LIVE. If no, it shows "--".

This status check can be done once on dropdown open rather than continuously polling all 20 tokens.

All 20 tokens are always selectable regardless of LIVE status. Selecting a token without active data loads the page with "--" values and "Awaiting venue data" states.

## Step 5: Place the Unified Selector

The ILU-20 selector component should appear in one consistent location across all pages. There are two approaches:

### Option A (Recommended): In the shared header bar

Place the selector in `dashboard-header.tsx` so it appears on every page in the same position. This is the cleanest approach because the user always knows where to find it.

```
STRATALINK LABS LIQUIDITY INTELLIGENCE TERMINAL    [BTC Bitcoin · RESERVE ▾]    ● LIVE    [🔔] [⚙] [→]
```

The selector sits in the header bar, right of centre, before the LIVE indicator. It is always visible and always accessible regardless of which page tab is active.

### Option B: In a sub-header strip below the platform tabs

Add a thin strip below the platform tabs that shows on every page:

```
TILT    STRATA AI    RCL    ALERTS
─────────────────────────────────────────────────────
TOKEN: [BTC Bitcoin · RESERVE ▾]    14 VENUES · 17 TSLE FEEDS    ● LIVE
```

Use whichever approach integrates more cleanly with the existing layout. Option A is preferred because it does not add vertical space.

## Step 7: Handle Token Changes

When the user selects a new token from the dropdown:

1. The global context updates immediately.
2. Every page that is currently mounted re-renders with the new token.
3. Data fetching for the new token begins immediately (new L5F snapshot request, new RCL data request, etc.).
4. While new data is loading, the page should show a brief loading state or transition rather than showing stale data from the previous token.
5. If the new token has no active venue data, all data-dependent components show "--" or "Awaiting venue data" states.
6. The token selection persists across tab navigation. If you select SOL on TILT and then click to STRATA AI, the STRATA AI page shows SOL data.

## Step 8: Backend Considerations

The backend endpoints currently accept a token or pair parameter. Verify that:

1. The L5F snapshot endpoint accepts any ILU-20 pair, not just BTC-USD, ETH-USD, and SOL-USD.
2. The RCL API endpoints accept the selected token as a parameter.
3. The PoLi engine can compute scores for any token the venue relays provide data for.
4. For tokens without active venue data, the endpoints return a clear "no data" response (not an error) so the frontend can display the appropriate empty state.

If any endpoint is hardcoded to only accept BTC, ETH, or SOL, update it to accept any ILU-20 token pair. The venue relay system may only return data for a subset of the 20 tokens, but the API layer should accept any of them and return whatever data is available (or an empty response for unmonitored tokens).

- Do not modify the L5F computation engine or PoLi scoring logic.
- Do not modify the TSLE buffer architecture.
- Do not change any API response formats. Only ensure the endpoints accept the ILU-20 pairs as valid input parameters.
