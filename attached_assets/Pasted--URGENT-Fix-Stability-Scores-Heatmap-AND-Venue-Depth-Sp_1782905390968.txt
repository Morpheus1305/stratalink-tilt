## The Fix

### Step 1: Find where the stability score is computed

Search the codebase for where the stability value (the number shown
inside each heatmap square, and the STATUS column in the venue table)
is calculated. Look for:
- A field called `stability`, `stabilityScore`, `venueStability`, or
  similar
- Any logic that checks `regulated`, `isRegulated`, `venue.regulated`,
  or the venue's regulatory status
- The values 80 and 0 being assigned based on a boolean condition

This is likely in one of these files:
- server/services/poliEngine.ts
- server/routes/rcl.ts
- A venue scoring utility
- The PoLi/PoMI page component itself

### Step 2: Replace the binary logic with a composite score

DELETE any logic that looks like:
  if (venue.regulated) { stability = 80 } else { stability = 0 }
  or
  stability = venue.regulated ? 80 : 0

REPLACE IT with this composite calculation:

```typescript
function computeVenueStability(venue: {
  depth: number;      // in dollars, e.g. 72000000
  spread: number;     // in bps, e.g. 4.2
  regulated: boolean;
}, maxDepth: number): number {
  // Depth component (0-100): venue depth relative to the largest venue
  const depthScore = maxDepth > 0
    ? Math.min(100, (venue.depth / maxDepth) * 100)
    : 0;

  // Spread component (0-100): tighter spread = higher score
  const spreadScore = Math.max(0, 100 - (venue.spread * 5));

  // Regulated bonus: 100 if regulated, 0 if not
  const regulatedBonus = venue.regulated ? 100 : 0;

  // Composite: depth 40%, spread 35%, regulated 25%
  const composite = (0.40 * depthScore) + (0.35 * spreadScore) + (0.25 * regulatedBonus);

  return Math.round(composite);
}
```

Call this function for every venue, passing the maximum depth across all
active venues as `maxDepth`.

### Step 3: Apply the composite score to BOTH panels

**Heatmap square colour AND score label:**
  composite >= 60 → GREEN (#00E676), label = composite value
  composite 35-59 → AMBER (#FFB300), label = composite value
  composite < 35  → RED (#FF5252), label = composite value

**Venue Depth & Spread Monitor STATUS column:**
  composite >= 60 → GREEN dot
  composite 35-59 → AMBER dot
  composite < 35  → RED dot

Both panels MUST use the same composite score so they always agree.

### Step 4: Update the legend count

The STABLE / STRESSED / CRITICAL legend below the heatmap must count
venues using the same thresholds:
  STABLE = count of venues with composite >= 60
  STRESSED = count of venues with composite 35-59
  CRITICAL = count of venues with composite < 35
